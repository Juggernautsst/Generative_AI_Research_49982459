# Generative AI Research

This repository stores research materials for the project "Using Generative AI Tools - Boon or Bane."

## Project Structure

- `literature_review/`: literature review documents such as journal articles, conference papers, books, and news articles.
- `data/raw_restricted/`: raw research data that may contain sensitive or identifiable information.
- `data/processed/`: cleaned or de-identified data used for analysis.
- `instruments/`: survey questions, interview protocols, consent forms, and information sheets.
- `analysis/quantitative/`: survey analysis scripts, CSV files, Excel files, and quantitative reports.
- `analysis/qualitative/`: interview coding notes, thematic analysis files, and qualitative insights.
- `visualisations/`: figures, charts, and other visual outputs.
- `reports/`: draft proposals, conference papers, and final reports.
- `admin/`: meeting notes and project management documents.

## Naming Convention

Use descriptive file names with a consistent format:

`YYYYMMDD_GAIResearch_DataType_Description_Version.ext`

Example:

`20260427_GAIResearch_SurveyData_Cleaned_v01.csv`

## Collaboration Guidelines

1. Create a new branch before making changes.
2. Use meaningful branch names, such as `add-survey-data` or `update-readme`.
3. Write commit messages that clearly describe the change.
4. Open a pull request before merging changes into `main`.
5. Do not upload identifiable or sensitive human research data to a public repository.

## Data Access

Sensitive files such as consent forms, identifiable survey responses, interview transcripts, and media files should be stored with restricted access. De-identified or non-sensitive files can be shared with collaborators for analysis and review.
